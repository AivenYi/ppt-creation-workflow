# 🏭 PPT Creation Workflow Skill — 双阶段流水线

**AI PPT智能创建工作流**：这是一个**双阶段流水线**，不是一次生成搞定的。

本skill不直接生成PPT，而是生产两套专业化提示词，让适合的工具做适合的事。

## 第一阶段：需求收集（向用户提问）

使用 AskUserQuestion 工具，按顺序向用户收集以下信息。

**设计原则**：
- 所有问题必须支持"其他/自定义"选项，用户可自由输入
- 选项只是快速选择的快捷方式，不是限制
- 优先使用 text 类型问题，让用户自由描述
- 每轮最多问2-3个问题，避免信息过载
- 用户回答模糊时，主动追问具体化

### 第一轮：基础信息（必须确认）

| 问题 | 说明 |
|------|------|
| PPT的主题是什么？ | 自由输入，问具体名称 |
| 这份PPT的用途是什么？ | 教学课件/商业提案/产品发布/工作汇报等 |
| 目标受众是谁？ | 零基础学员/管理者/投资者/专业人士等，可细化 |
| 演示时长大概多久？ | 直接输入分钟数 |

### 第二轮：风格 & 内容需求（必须确认）

| 问题 | 说明 |
|------|------|
| 偏好哪种视觉风格？ | 简约现代/科技感/中国风/卡通/商务/像素风/电影感等 |
| 是否有品牌色或主色调？ | 无要求或具体色号 |
| 插图/配图风格偏好？ | AI生成配图/真实摄影/扁平插画/3D渲染等 |

**关键追问**：用户说"不知道选什么风格" → 提供3-5个风格案例描述让用户选

### 第三轮：内容深度确认（必须详细追问）

**这是最关键的步骤**。不要自己拍脑袋填充内容，必须追问用户：

```
┌─────────────────────────────────────────────┐
│  ！必须问的问题（否则不能进入下一步）        │
├─────────────────────────────────────────────┤
│  1. 你有没有特定的内容想要包含？             │
│     （比如：特定案例、特定工具、特定方法等）  │
│                                              │
│  2. 你主要想介绍哪些工具/平台？              │
│     （哪个是重点？哪些一笔带过？）            │
│                                              │
│  3. 大纲方向有没有偏好？                     │
│     （比如按时间线/按工具/按场景分类）        │
│                                              │
│  4. 有没有注意事项/风格限制/边界条件？        │
│     （比如某些内容已经结束、某些功能要强调）  │
└─────────────────────────────────────────────┘
```

### 第四轮：页码估算 & 平台确认

| 问题 | 说明 |
|------|------|
| 大概需要多少页？ | 通常每2-3分钟1页估算 |
| Gemini生成内容后，用哪个工具做PPT？ | 豆包DOCPPTA/Gamma/Tome等 |

## 第二阶段：内容策划（产出完整大纲）

根据收集的信息，按照以下结构组织PPT内容：

### 大纲结构模板

```
封面页（1页）
目录页（1页）
正文内容（分X个部分，每部分N页）
├─ 部分一：[核心主题]
│   ├─ 引入/概念
│   ├─ 详细内容
│   └─ （如有）案例/实操
├─ 部分二：[核心主题]
│   └─ ...
└─ ...
总结页（1页）
作业/资源页（1页）
结课页（1页）
```

### 内容填充原则

1. **每页一个核心信息点** - 避免信息过载
2. **标题即结论** - 用结论做标题，不用"概述"、"背景"等空洞词
3. **善用对比结构** - 前后对比、优劣势对比、工具对比
4. **实操页面必须标注⏱时间** - 用户演讲时知道节奏
5. **插图描述要具体可生成** - 含色彩、构图、风格关键词

## 第三阶段：生成 Gemini 提示词

这是第一阶段的核心产出。将完整大纲包装成语Gemini可以理解的专业提示词。

### Gemini提示词结构模板

```markdown
# 角色设定
你是一位顶尖的{领域}讲师与PPT设计专家。请根据以下大纲，生成一份结构清晰、内容充实的PPT课件文字内容。

# 基础信息
- 主题：{PPT主题}
- 用途：{用途}
- 受众：{受众}
- 时长：{时长}
- 页数：{页数}

# 视觉风格要求
- 整体风格：{用户选择的风格}
- 配色：主色/辅色/背景色
- 排版规范：双栏对比/三卡片/流程图等

# 内容大纲（共{页数}页，{部分数}部分）
{完整大纲}

# 输出要求
1. 每页按格式：标题 | 核心文字 | 排版建议 | 插图描述
2. 文字精炼，标题即结论
3. 插图描述要具体（含色彩/构图/风格）
4. 实操页面标注⏱时间
```

## 第四阶段：根据 Gemini 输出生成豆包提示词

**这是第二阶段的核心产出。** 用户将Gemini输出的内容发回来，我将其包装成豆包可用的PPT生成提示词。

### 豆包提示词结构模板

```markdown
# 角色设定
你是一位顶尖的PPT设计专家与{领域}讲师。请根据以下详细内容，生成一份具有高度设计感、结构清晰的演示文稿。

# 基础信息
- 主题：{PPT主题}
- 副标题：{副标题}
- 主讲人：[姓名]
- 用途：{用途}
- 受众：{受众}
- 时长：{时长}
- 页数：{页数}

# 视觉风格（核心要求）
- 主风格：{具体风格描述}
- 配色方案：主色+辅色+强调色（含HEX代码）
- 字体规范：标题/正文/注释
- 视觉元素：{具体icon/装饰描述}
- 排版规范：双栏/三栏/四宫格等

# 详细内容（共{页数}页）

按照Gemini输出逐页填入，每页格式：
### 第N页：[页面标题]
- **核心文字**：{精炼要点}
- **排版建议**：{具体排版方式}
- **配图描述**：{具体插图描述}

# 输出要求
1. 每页4要素：标题 | 核心文字 | 排版 | 配图
2. 文字精炼，标题即结论
3. 配图描述具体可生成
4. 实操页面标注⏱时间
```

|------|------|
| Gemini 访问 | 需要能访问 [gemini.google.com](https://gemini.google.com)（免费）。**中国大陆用户可能需要代理/VPN** |

| WorkBuddy | 本 skill 在 WorkBuddy 中运行，依赖 `AskUserQuestion` 工具 |

> ⚠️ **关于代理/VPN**：本 skill 本身**不发起任何网络请求**，不受代理环境影响。但你在"第3步将提示词复制发给 Gemini"时，Gemini 在中国大陆可能无法直连。如果遇到 Gemini 不可用的情况，可参考下方备选方案。

### 备选方案（Gemini不可用时）

| 方案 | 替代工具 | 效果 |
|------|----------|------|
| A | DeepSeek / Claude / ChatGPT | 同样可以生成 PPT 详细内容，效果接近 |
| B | 直接把大纲发给豆包 | 跳过双阶段，单次生成（页数较少时推荐） |
| C | 先让豆包生成初稿 → 再发给 Gemini 优化 | 缩短 Gemini 使用时间 |

## 常见问题处理

### Q：我能不能直接让豆包生成，不走Gemini？
→ 可以，但如果内容量大（30页+）、对配图/排版有严格要求，走双阶段效果更好。Gemini擅长内容生成，豆包擅长PPT渲染。

### Q：Gemini输出后我需要等多久？
→ 通常几分钟，50页内容输出后直接复制回来给我即可。

### Q：我不确定要什么风格/内容怎么办？
→ 没关系，我会先问几个简单问题帮你理清思路。

### Q：这个skill支持Gamma/Tome吗？
→ 支持。在需求收集阶段告诉我使用哪个平台，我会针对不同平台调整提示词格式。

### Q：代理/VPN 会影响这个 skill 吗？
→ **不会。** skill 本身只在本地运行，不发起任何网络请求。只有你在浏览器中访问 Gemini 时才需要网络，那是完全独立于 skill 的操作。WorkBuddy 自带代理设置（如 `HTTP_PROXY`）也不会影响 skill 的工作流。

### Q：如果 Gemini 就是打不开怎么办？
→ 使用备选方案 A 或 B。本 skill 生成的提示词结构通用，可以发给任何大语言模型（DeepSeek / Claude / ChatGPT / Kimi 等），只需要把提示词中的 "Gemini" 当作 "AI 助手" 即可。
